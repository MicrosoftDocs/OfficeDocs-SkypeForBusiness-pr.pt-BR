## <a name="configuring-skype-for-business-server-to-be-a-partner-application-for-exchange-server"></a><span data-ttu-id="47854-113">Configurando o Skype para Business Server seja um aplicativo parceiro do Exchange Server</span><span class="sxs-lookup"><span data-stu-id="47854-113">Configuring Skype for Business Server to be a Partner Application for Exchange Server</span></span>

<span data-ttu-id="47854-114">A maneira mais fácil de configurar Skype para Business Server seja um aplicativo parceiro com o Exchange Server 2016 ou Exchange Server 2013 é executar o script Configure-EnterprisePartnerApplication.ps1, um script do Windows PowerShell que acompanha o Exchange Server.</span><span class="sxs-lookup"><span data-stu-id="47854-114">The easiest way to configure Skype for Business Server to be a partner application with Exchange Server 2016 or Exchange Server 2013 is to run the Configure-EnterprisePartnerApplication.ps1 script, a Windows PowerShell script that ships with Exchange Server.</span></span> <span data-ttu-id="47854-115">Para executar esse script, você deve fornecer a URL para o Skype para o documento de metadados de autenticação Business Server; Normalmente, esse será o nome de domínio totalmente qualificado do Skype para pool de servidores de negócios, seguido por /metadata/json/1 o sufixo.</span><span class="sxs-lookup"><span data-stu-id="47854-115">To run this script, you must provide the URL for the Skype for Business Server authentication metadata document; this will typically be the fully qualified domain name of the Skype for Business Server pool followed by the suffix /metadata/json/1.</span></span> <span data-ttu-id="47854-116">Por exemplo:</span><span class="sxs-lookup"><span data-stu-id="47854-116">For example:</span></span>

<span data-ttu-id="47854-118">Depois de configurar o aplicativo parceiro é recomendável que você parar e reiniciar os serviços de informações da Internet (IIS) em seus servidores de acesso de cliente e caixa de correio do Exchange.</span><span class="sxs-lookup"><span data-stu-id="47854-118">After configuring the partner application it is recommended that you stop and restart Internet Information Services (IIS) on your Exchange mailbox and client access servers.</span></span> <span data-ttu-id="47854-119">Você pode reiniciar o IIS usando um comando similar a esse, que reinicia o serviço no atl-exchange-001 do computador:</span><span class="sxs-lookup"><span data-stu-id="47854-119">You can restart IIS by using a command similar to this, which restarts the service on the computer atl-exchange-001:</span></span>
