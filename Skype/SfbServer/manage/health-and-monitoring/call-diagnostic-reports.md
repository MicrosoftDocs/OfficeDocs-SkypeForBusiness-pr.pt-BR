# <a name="call-diagnostic-reports-in-skype-for-business-server-2015"></a>Relatórios de Diagnóstico de Chamada no Skype for Business Server 2015
 
**Resumo:** Saiba mais sobre os vários usuários chamada relatórios de diagnóstico usada no Skype para Business Server 2015.
  
Os Relatórios de Diagnóstico de Chamada oferecem informações de resumo e dados de diagnóstico para sessões ponto a ponto e de conferência com falhas.
  
## <a name="in-this-section"></a>Nesta seção

- [Chamar o relatório de resumo de diagnóstico no Skype para Business Server 2015](summary-report.md) Fornece um resumo geral das sessões com falha de ponto a ponto e sessões de conferência. Sessões ponto a ponto são aquelas que envolvem somente dois participantes. Sessões de conferência envolvem três ou mais participantes.
    
- [Relatório de diagnóstico de atividade ponto a ponto no Skype para Business Server 2015](peer-to-peer-activity-diagnostic-report.md) Fornece uma visão geral de tendência das sessões com falha de ponto a ponto. Uma sessão ponto a ponto envolve somente dois participantes.
    
- [Relatório de diagnóstico de conferência em Skype para Business Server 2015](conference-diagnostic-report.md) Fornece uma visão geral de tendência de sessões de conferência com falha e modos de exibição de tendência para cada modalidade de conferência. Sessões de conferência envolvem no mínimo três participantes.
    
- [Relatório de falhas principais no Skype para Business Server 2015](top-failures-report.md) Fornece uma lista das falhas mais frequentes e suas tendências ao longo do tempo.
    
- [Relatório de distribuição com falha no Skype para Business Server 2015](failure-distribution-report.md) Fornece uma análise das sessões com falha.
    
- [Relatório de lista de falha no Skype para Business Server 2015](failure-list-report.md) Fornece informações detalhadas sobre os participantes individuais envolvidos em uma conferência com falhas.
    
- [Relatório de diagnóstico em Skype para Business Server 2015](diagnostic-report.md) Fornece informações de diagnóstico e solução de problemas (incluindo códigos de resposta SIP e cabeçalhos de diagnóstico e IDs) para sessões com falha.
