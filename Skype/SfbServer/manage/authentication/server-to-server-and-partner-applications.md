> <span data-ttu-id="abc22-116">Um realm é simplesmente um contêiner de segurança.</span><span class="sxs-lookup"><span data-stu-id="abc22-116">A realm is simply a security container.</span></span> <span data-ttu-id="abc22-117">Por padrão, o Skype for Business Server usa o seu domínio SIP padrão como seu território OAuth.</span><span class="sxs-lookup"><span data-stu-id="abc22-117">By default, Skype for Business Server uses your default SIP domain as its OAuth realm.</span></span> <span data-ttu-id="abc22-118">Namespaces SIP adicionais são adicionadas à lista Nome Alternativo da Entidade no certificado OAuth.</span><span class="sxs-lookup"><span data-stu-id="abc22-118">Additional SIP namespaces are added to the Subject Alternate Name list in the OAuth certificate.</span></span> 
  
<span data-ttu-id="abc22-119">O Skype for Business Server oferece suporte a três cenários de autenticação entre servidores.</span><span class="sxs-lookup"><span data-stu-id="abc22-119">Skype for Business Server supports three server-to-server authentication scenarios.</span></span> <span data-ttu-id="abc22-120">Com o Skype for Business Server, você pode:</span><span class="sxs-lookup"><span data-stu-id="abc22-120">With Skype for Business Server, you can:</span></span>

- <span data-ttu-id="abc22-123">Configurar a autenticação de servidor para servidor em um ambiente de várias instalações (ou seja, autenticação de servidor para servidor entre um servidor local e um componente do Office 365).</span><span class="sxs-lookup"><span data-stu-id="abc22-123">Configure server-to-server authentication in a cross-premises environment (that is, server-to-server authentication between an on-premises server and an Office 365 component).</span></span>

<span data-ttu-id="abc22-125">Também deve ser indicado que a autenticação servidor-a-servidor é opcional: se o Skype for Business Server não precisar se comunicar com outros servidores (como o Exchange), a autenticação de servidor para servidor pode ser totalmente ignorada.</span><span class="sxs-lookup"><span data-stu-id="abc22-125">It should also be pointed out that server-to-server authentication is optional: If Skype for Business Server does not need to communicate with other servers (such as Exchange) then server-to-server authentication can be skipped altogether.</span></span> <span data-ttu-id="abc22-126">Se a autenticação de servidor para servidor já estiver configurada para o Lync Server 2013 e outros aplicativos, não será necessário fazê-lo novamente para o Skype for Business Server.</span><span class="sxs-lookup"><span data-stu-id="abc22-126">If server-to-server authentication is already configured for Lync Server 2013 and other applications, there's no need to re-do it for Skype for Business Server.</span></span>

<span data-ttu-id="abc22-138">Para configurar a autenticação de servidor para servidor para uma implementação local do Skype for Business Server, você deve fazer duas coisas:</span><span class="sxs-lookup"><span data-stu-id="abc22-138">In order to configure server-to-server authentication for an on-premises implementation of Skype for Business Server, you must do two things:</span></span>
