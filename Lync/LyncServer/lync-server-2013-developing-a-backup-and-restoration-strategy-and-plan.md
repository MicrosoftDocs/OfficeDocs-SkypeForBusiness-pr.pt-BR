# <a name="developing-a-backup-and-restoration-strategy-and-plan-for-lync-server-2013"></a>Desenvolver uma estratégia de backup e restauração e planejar o Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Última modificação do tópico:** 2013-02-17_

A eficácia de suas operações de backup e restauração do Lync Server depende de sua estratégia de backup e restauração e o plano. Você deve estabelecer uma estratégia de backup e restauração do Lync Server que atenda à estratégia geral da sua organização e um plano abrangente e conciso para fazer backup de dados e configurações e, no caso de uma interrupção, um plano para restaurar o serviço.

Para a recuperação de desastres mais robusta de um pool de front-ends, use a topologia de recuperação de desastres do pool emparelhado introduzida no Lync Server 2013. Para obter mais informações, consulte [Planning for High Availability and Disaster Recovery in Lync Server 2013](lync-server-2013-planning-for-high-availability-and-disaster-recovery.md).
