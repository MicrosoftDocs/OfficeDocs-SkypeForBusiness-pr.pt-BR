# Planejamento de conectividade de PSTN no Lync Server 2013

 

_**Tópico modificado em:** 2012-09-21_

Uma solução de VoIP em nível empresarial deve fornecer chamadas de e para PSTN (rede telefônica pública comutada), sem redução de QoS (Qualidade de Serviço). Os usuários que fazem e recebem chamadas não devem perceber a tecnologia subjacente: da perspectiva do usuário, uma chamada entre a infraestrutura do Enterprise Voice e a PSTN deve parecer apenas um telefonema normal.

O Lync Server 2013 fornece uma conectividade PSTN confiável e escalável usando as seguintes opções:

  - **Troncos SIP** para um provedor de serviço de telefonia da Internet (ITSP)

  - **Conexões SIP diretas** com um gateway PSTN

  - **Conexões SIP diretas** com um PBX

Dependendo de seu tamanho, cobertura geográfica e infraestrutura de voz existente, uma determinada empresa pode usar uma, duas ou até mesmo três opções em vários locais.
