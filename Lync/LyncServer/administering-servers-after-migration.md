# <a name="administering-servers-after-migration"></a>Administrar servidores após migração

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Tópico da última modificação:** 2012-09-29_

Em geral, você deve usar a ferramenta administrativa que corresponde à versão do servidor que você deseja gerenciar. Não é possível instalar o Lync Server 2010 e as ferramentas administrativas do Lync Server 2013 no mesmo computador. Além disso, o painel de controle do Lync Server 2013 não é instalado automaticamente em cada servidor. Para instalar o painel de controle do Lync Server 2013, siga o procedimento dentro do tópico [instalar as ferramentas administrativas do Lync server 2013](lync-server-2013-install-lync-server-administrative-tools.md) na documentação de implantação.

<div>


> [!IMPORTANT]  
> Depois que um pool piloto do Lync Server 2013 é implantado, você não pode usar o construtor de topologias do Lync Server 2010 ou o painel de controle do Lync Server 2010 para gerenciar qualquer recurso do Lync Server 2013. Você deve usar as ferramentas do Lync Server 2013 para gerenciar os recursos do Lync Server 2013 e do Lync Server 2010.
