# <a name="configure-automatic-client-sign-in-to-use-the-director-in-lync-server-2013"></a>Configurar a entrada automática de cliente para usar o diretor no Lync Server 2013

</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Última modificação do tópico:** 2012-09-08_

Ao implantar um Lync Server 2013, diretor ou um pool de diretores, recomendamos que você use o logon automático do cliente como uma prática recomendada. Para obter detalhes sobre como configurar servidores DNS para entrada automática de cliente, consulte [DNS Requirements for Automatic Client Sign-in in Lync Server 2013](lync-server-2013-dns-requirements-for-automatic-client-sign-in.md) na documentação de planejamento.

Se você já implantou a Entrada Automática do Cliente, consulte as seções a seguir para configurá-la em seu(s) Diretor(es).

<div>

## <a name="single-director-instance"></a>Instância de um único Diretor

Se você já tiver a entrada automática de cliente implantada e estiver apontando para um servidor front-end ou um pool de front-ends, será necessário alterar o registro SRV DNS para apontar para o diretor.

</div>

<div>

## <a name="director-pool"></a>Pool de Diretor

Se você já tiver a entrada automática de cliente implantada e estiver apontando para um servidor front-end ou um pool de front-ends, será necessário alterar o registro SRV DNS para apontar para o pool de diretores.

</div>
