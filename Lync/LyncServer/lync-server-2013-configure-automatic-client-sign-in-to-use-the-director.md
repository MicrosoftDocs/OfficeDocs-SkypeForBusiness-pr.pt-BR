# <a name="configure-automatic-client-sign-in-to-use-the-director-in-lync-server-2013"></a>Configurar o Sign-In automático do cliente para usar o diretor no Lync Server 2013

<div data-xmlns="http://www.w3.org/1999/xhtml">

<div class="topic" data-xmlns="http://www.w3.org/1999/xhtml" data-msxsl="urn:schemas-microsoft-com:xslt" data-cs="https://msdn.microsoft.com/">

<div data-asp="https://msdn2.microsoft.com/asp">



</div>

<div id="mainSection">

<div id="mainBody">

<span> </span>

_**Última modificação do tópico:** 2012-09-08_

Quando você implanta um Lync Server 2013, diretor ou um pool de diretores, recomendamos que você use o cliente automático Sign-In como prática recomendada. Para obter detalhes sobre como configurar servidores DNS para entrada automática de cliente, consulte [DNS Requirements for Automatic Client Sign-in in Lync Server 2013](lync-server-2013-dns-requirements-for-automatic-client-sign-in.md) na documentação de planejamento.

Se você já implantou a Entrada Automática do Cliente, consulte as seções a seguir para configurá-la em seu(s) Diretor(es).

<div>

## <a name="single-director-instance"></a>Instância de um único Diretor

Se você já tiver o cliente Sign-In implantado e ele estiver apontando para um servidor front-end ou um pool de front-ends, será necessário alterar o registro SRV DNS para apontar para o diretor.

</div>

<div>

## <a name="director-pool"></a>Pool de Diretor

Se você já tiver o cliente Sign-In implantado e ele estiver apontando para um servidor front-end ou um pool de front-ends, será necessário alterar o registro SRV DNS para apontar para o pool de diretores.

</div>
