# <a name="microsoft-staffhub-to-be-retired"></a>O Microsoft StaffHub será desativado

A partir 1 de outubro de 2019, o Microsoft StaffHub será desativado. Estamos criando recursos StaffHub no Microsoft Teams. Hoje, o Teams inclui o aplicativo Shifts para o gerenciamento de cronogramas e recursos adicionais serão implantados com o tempo.

Essas alterações fazem parte dos nossos esforços contínuos para capacitar todos os funcionários com o Microsoft 365. Com os recursos para os Firstline Workers agora no Teams todos os funcionários da sua organização poderão usar o Teams para simplificar o trabalho, colaborar com colegas e acessar informações e experiências para ajudá-los a realizar um trabalho melhor.

## <a name="frequently-asked-questions"></a>Perguntas frequentes

### <a name="about-staffhub-retirement"></a>Sobre a desativação do StaffHub

#### <a name="when-will-staffhub-be-retired"></a>Quando o StaffHub será desativado?

A partir do meio de 2019, o Microsoft StaffHub não estará mais disponível para novos locatários e você não poderá mais acessar o aplicativo do inicializador de aplicativos do Office 365 em locatários existentes. Os usuários atuais ainda poderão usar o serviço até 1 de outubro de 2019, mas nenhum recurso adicional será disponibilizado.

#### <a name="what-will-happen-once-staffhub-is-retired"></a>O que acontecerá quando StaffHub for desativado?

O Microsoft StaffHub deixará de funcionar para todos os usuários em 1 de outubro de 2019. Todas as pessoas que tentarem abrir o aplicativo receberão uma mensagem direcionando-as para o download do Teams. Se o usuário tiver uma licença ativa que inclua o Teams e a organização fez as etapas necessárias para habilitar o Teams, os dados e as funcionalidades principais do StaffHub estarão disponíveis na sua transição para o Teams.

Entre abril e outubro de 2019, os usuários do StaffHub receberão notificações no aplicativo incentivando-os a usar o Teams.

#### <a name="will-teams-offer-all-the-functionality-currently-offered-in-staffhub"></a>O Teams possui todas as funcionalidades oferecidas pelo StaffHub?

Em janeiro de 2019 o Teams começou a ofertar o Shifts que antes estava disponível no Microsoft StaffHub. Esse recurso permite que os gerentes planejem e distribuam os cronogramas de turnos e que os membros da equipe tenham a oportunidade de revisar a escala e solicitar alterações através de dispositivos móveis. Para saber mais, confira [Gerenciar o aplicativo Shifts para sua organização no Teams](manage-the-shifts-app-for-your-organization-in-teams.md).

Temos o compromisso de oferecer uma experiência de usuário superior e trazer os recursos exclusivos do StaffHub para o Teams. Para obter mais informações sobre quais recursos estarão disponíveis no Teams, confira [Recursos para Firstline Worker em breve no Teams](firstline-worker-features-coming-to-teams.md).

#### <a name="what-do-i-need-to-do-to-prepare-for-this-change"></a>O que preciso fazer para me preparar para essa alteração?

Para saber como se preparar para a transição do StaffHub para o Shifts no Teams, vá para [Preparar-se para mudar as equipes do StaffHub para o Shifts](move-staffhub-teams-to-shifts-in-teams.md#prepare).

#### <a name="why-cant-users-in-my-organization-access-their-staffhub-team-from-shifts-in-teams"></a>Por que os usuários em minha organização não têm acesso à equipe do StaffHub a partir do Shifts no Teams?

As equipes criadas com o StaffHub só podem ser visualizadas e gerenciadas no StaffHub. Depois de uma equipe do StaffHub ser movida para o Teams, os usuários poderão exibir suas agendas no Shifts. Incentivamos você a comunicar essa mudança antecipadamente e familiarizar os usuários existentes com o Teams.

### <a name="about-shifts"></a>Sobre o Shifts

#### <a name="what-is-shifts"></a>O que é o Shifts?

O aplicativo Shifts no Teams é uma ferramenta de gerenciamento de agendamento que mantem os Firstline Workers conectados e em sincronia. Os gerentes criam, atualizam e gerenciam cronogramas de turnos para equipes, e os membros da equipe podem ver suas escalas, ver quem mais está agendado para o dia, solicitar a troca ou oferecer um turno e solicitar folga, tudo isso através de dispositivos móveis.  

#### <a name="which-plans-is-shifts-available-in"></a>Em quais planos o Shifts está disponível?

O Shifts está disponível como parte do Teams nos seguintes planos, sem custo adicional:

- Office 365 Business Premium 
- Office 365 Enterprise F1, E1, E3, E5
- Office 365 for Education
- Microsoft 365 F1, E1, E3, E5

Para saber mais, confira [Disponibilidade do Shifts](manage-the-shifts-app-for-your-organization-in-teams.md#availability-of-shifts).

#### <a name="how-do-i-get-shifts"></a>Como faço para obter o Shifts? 

Se você já estiver em um plano que inclua o Shifts, você pode começar a usar o Shifts hoje como parte do Teams. Confira [Introdução ao Shifts](https://support.office.com/pt-BR/article/get-started-in-shifts-5f3e30d8-1821-4904-be26-c3cd25a497d6).

#### <a name="which-devices-or-platforms-is-shifts-available-on"></a>Em quais dispositivos ou plataformas o Shifts está disponível?

O Shifts está disponível no cliente Web, no cliente da área de trabalho e nos clientes móveis (iOS e Android) do Teams.
 
#### <a name="are-there-differences-between-the-shifts-desktop-and-web-apps-and-the-mobile-ios-and-android-apps"></a>Há diferenças entre os aplicativos Shifts área de trabalho e Web e os aplicativos móveis (iOS e Android)?

Para os gerentes, os aplicativos da área de trabalho e da Web são otimizados para criar equipes, adicionar e publicar escalas e compartilhar conteúdo com a equipe. Os gerentes geralmente usam o aplicativo móvel para aprovar solicitações, enviar mensagens rápidas para a equipe, ligar para um membro da equipe e colaborar em qualquer lugar. 

Para os membros da equipe, o aplicativo móvel é otimizado para ver a escala dele, ver com quem estão trabalhando em um dia, registrar entrada e saída, definir a disponibilidade, solicitar folga, trocar ou oferecer turnos e abrir turnos. Há diferenças no que os usuários podem fazer nos aplicativos da área de trabalho e na Web em comparação com os aplicativos móveis. Estamos trabalhando na paridade dos recursos.

#### <a name="are-there-differences-between-the-shifts-ios-app-and-the-android-app"></a>Há diferenças entre o aplicativo Shifts iOS e Android?

A principal funcionalidade é a mesma entre os aplicativos Shifts iOS e Android. Existem pequenas diferenças de interface do usuário entre os dois aplicativos para alinhar aos padrões de design recomendados.

#### <a name="what-browsers-does-shifts-work-on"></a>Em quais navegadores o Shifts funciona?

O Shifts funciona em todos os navegadores compatíveis com o Teams, que inclui as versões mais recentes do Microsoft Edge, Internet Explorer, Google Chrome e Mozilla Firefox. Para saber mais, confira [Obter clientes para o Teams.](../../get-clients.md)

#### <a name="what-languages-does-shifts-support"></a>O Shifts oferece suporte a quais idiomas?

O Shifts está disponível nos seguintes idiomas: chinês simplificado, inglês (EUA), espanhol, russo, japonês, francês, Português (Brasil), alemão, coreano, italiano, chinês tradicional, holandês, turco, Sueco, dinamarquês e norueguês. Planejamos adicionar suporte para outros idiomas em breve.  

#### <a name="where-is-shifts-data-stored"></a>Onde os dados são armazenados?

O dados do Shifts são armazenados no Azure, em datacenters na América do Norte e na Europa Ocidental. O conteúdo do SharePoint aproveita os grupos do Office 365 e usa o mesmo armazenamento e suporte.

Para saber mais, confira [Localização dos dados do Shifts](manage-the-shifts-app-for-your-organization-in-teams.md#location-of-shifts-data).

#### <a name="where-can-i-learn-more-about-shifts"></a>Onde posso saber mais sobre o Shifts?

Para obter informações sobre o produto, vá para [https://products.office.com/microsoft-teams/staff-scheduling-software](https://products.office.com/microsoft-teams/staff-scheduling-software).

Para obter diretrizes de administrador, confira[Gerenciar o aplicativo Shifts para sua organização no Teams](manage-the-shifts-app-for-your-organization-in-teams.md)

Para obter ajuda para usuários finais, acesse a [Ajuda do Shifts para Firstline Workers](https://support.office.com/article/apps-and-services-cc1fba57-9900-4634-8306-2360a40c665b).

### <a name="switch-from-staffhub-to-shifts-in-teams"></a>Alternar do StaffHub para o Shifts no Teams.

#### <a name="what-licenses-do-i-need-for-teams"></a>De quais licenças eu preciso no Teams?

Cada usuário deve ter uma licença ativa do Microsoft 365 ou do Office 365 de um [plano elegível](#which-plans-is-shifts-available-in) e deve ter uma licença do Teams. Para obter mais informações, confira [Migrar suas equipes do StaffHub para o Shifts no Teams](move-staffhub-teams-to-shifts-in-teams.md).

#### <a name="how-do-i-move-my-organizations-staffhub-teams-to-teams"></a>Como faço para migrar as equipes do StaffHub da minha organização para o teams?

Para obter instruções detalhadas sobre como fazer a transição do StaffHub para o Teams, confira [Mova as suas equipes do Microsoft StaffHub para o Shifts no Teams](move-staffhub-teams-to-shifts-in-teams.md).

#### <a name="how-do-i-enable-teams-in-my-organization"></a>Como faço para habilitar o Teams na minha organização?

Por padrão, o Microsoft Teams está habilitado para todas as organizações. Você atribui licenças do Teams aos usuários para controlar o acesso individual ao Teams. Para obter mais informações, confira [Habilitar o Teams em sua organização](../../Office-365-set-up.md) e [Gerenciar o acesso ao Teams](../../user-access.md).

#### <a name="what-if-i-currently-have-skype-for-business-enabled-in-my-organization-but-not-teams"></a>E se eu tiver o Skype for Business habilitado na minha organização, mas não o Teams?

O Teams oferece suporte a coexistência com o Skype for Business. Para saber mais, confira [Compreender a coexistência e a interoperabilidade do Teams e do Skype for Business](../../teams-and-skypeforbusiness-coexistence-and-interoperability.md) e [Orientações de migração e interoperabilidade para organizações que usam o Teams em conjunto com o Skype for Business](../../migration-interop-guidance-for-teams-with-skype.md).

#### <a name="does-each-user-need-to-have-an-office-365-account-or-azure-active-directory-azure-ad-account"></a>Cada usuário deve ter uma conta do Office 365 ou uma conta do Azure Active Directory (Azure AD)?

Sim, cada gerente e membro da equipe que usa o Teams e o Shifts dentro do Teams deve ter uma conta do Office 365 (Azure AD) e deve ter uma licença válida atribuída.
 
#### <a name="in-staffhub-i-could-add-a-special-prefix-to-all-office-365-groups-that-were-created-by-staffhub-teams-can-i-do-the-same-in-teams"></a>No StaffHub, posso adicionar um prefixo especial a todos os Grupos do Office 365 criados por equipes do StaffHub. Posso fazer o mesmo no Teams?

Você não terá o mesmo recurso quando migrar do StaffHub para o Teams. Se você precisar aplicar a consistência de nomes a todos os Grupos do Office 365, é possível usar a[Política de nomeação de grupos do Office 365](https://docs.microsoft.com/office365/admin/create-groups/groups-naming-policy).

#### <a name="will-team-names-be-the-same-in-teams"></a>Os nomes da equipe serão os mesmos no Teams?

É possível que o nome de uma equipe seja diferente. Nomes de equipe no Teams usam o nome do Grupo do Office 365. Se o nome da equipe do StaffHub for diferente do nome do Grupo do Office 365 correspondente, você verá um nome diferente quando alternar para o Teams.

#### <a name="what-happens-to-team-membership-when-a-staffhub-team-is-moved-to-teams"></a>O que acontece com os membros da equipe quando uma equipe do StaffHub é movida para o Teams?

Os membros da equipe são mantidos quando uma equipe do StaffHub é movida para o Teams. Os números de telefone dos membros da equipe não serão exibidos no Shifts. Se o número de telefone no StaffHub corresponder ao número do telefone no Azure AD, ele será armazenado no Azure AD para essa conta de usuário. 

Recomendamos que você examine seus registros de equipe existentes no StaffHub e, quando possível alinhe os números de telefone a identidades relevantes do Azure AD.
 
#### <a name="what-happens-to-staffhub-team-members-who-arent-provisioned-or-whose-status-is-inactive"></a>O que acontece com os membros da equipe do StaffHub que não têm provisionamento ou cujo status está inativo?

A escala do usuário será movida do Shifts, no entanto, o usuário não será membro da equipe ou de um Grupo do Office 365.

Para contas inativas que não estão vinculadas a uma conta de usuário do Azure AD, um gerente pode garantir que o usuário seja adicionado como um membro da equipe, adicionando um endereço de email válido do Office 365 na página de membros da equipe do StaffHub para que o usuário esteja vinculado à conta do Azure AD correta na organização.

Outra opção é que os administradores executem os cmdlets **Add-StaffHubMember** e **Remove-StaffHubUser** para remover uma conta não provisionada de uma equipe do StaffHub e adicionar o usuário novamente à equipe com o seu UPN.

Recomendamos que você verifique se todos os usuários em sua organização possuam uma licença adequada para minimizar a interrupção e o acesso aos serviços.
 
#### <a name="what-happens-to-chats-in-staffhub"></a>O que acontece com os chats do StaffHub?

Para ajudar os usuários em transição, planejamos migrar o histórico dos dados do chat do StaffHub para o Teams. A quantidade do histórico do chat que estará disponível no Teams dependerá de quanto tempo você configurar sua organização para a transição. É recomendável executar uma ação no máximo em 30 de junho de 2019 para fornecer tempo suficiente para a transição de 90 dias de dados até 1 de outubro de 2019.

Certifique-se de que a criação de grupo do Office 365 está habilitada no centro de administração do Microsoft 365 antes que a transição comece, pois isso é essencial para que possamos migrar seus dados do chat do StaffHub para o Teams. Se você não ativar esse serviço até 30 de junho de 2019, resultará em uma data de migração posterior ou menos dados de chat sendo copiados.

#### <a name="what-happens-to-files-that-have-been-uploaded-to-staffhub"></a>O que acontece com os arquivos que foram carregados para o StaffHub?

Você pode mover os arquivos que suas equipes precisam do StaffHub para o Teams. Os arquivos que você move são armazenados na pasta Geral do site da equipe no SharePoint Online. No Teams, os usuários podem acessar os arquivos na guia **Arquivos** no canal Geral da equipe. Para obter mais informações, confira [Mova as suas equipes do Microsoft StaffHub para o Shifts no Teams](move-staffhub-teams-to-shifts-in-teams.md).

#### <a name="can-i-export-shift-schedules-from-shifts"></a>Posso exportar escalas de turnos do Shifts?
  
É possível exportar as informações de agendamento de turnos para um arquivo do Excel a partir dos aplicativos Shifts área de trabalho e Web. É possível importar essas informações para os sistemas e ferramentas da sua organização.

#### <a name="how-can-i-recover-a-schedule-if-i-accidentally-deleted-a-team"></a>Como posso recuperar uma escala se eu tiver excluído acidentalmente uma equipe?

A exclusão de uma escala está vinculada à exclusão de uma equipe. Se você recuperar uma equipe, a escala não será recuperada.

#### <a name="what-powers-the-file-sharing-capability-in-teams-where-are-files-stored"></a>O que permite o recurso de compartilhamento de arquivos no Teams? Onde os arquivos são armazenados?

O recurso de compartilhamento de arquivo, que permite que um gerente distribua arquivos para os membros da equipe, está conectado ao back-end do SharePoint Online. Quando uma equipe é criada, um grupo do Office 365 é automaticamente provisionado para essa equipe em segundo plano.

Para saber mais, confira [Como o SharePoint Online e o OneDrive for Business interagem com o Teams](../../SharePoint-OneDrive-interact.md).
 
#### <a name="is-there-integration-between-shifts-and-the-outlook-calendar-can-i-export-a-schedule-to-a-calendar"></a>Há integração entre to Shifts e o calendário do Outlook? Posso exportar uma escala para um calendário?
 
Firstline Workers podem ver toda a escala de trabalho em um piscar de olhos. Vamos dar suporte à integração nativa com o calendário do Outlook como parte do nosso roteiro. Enquanto isso, use o Microsoft Flow para configurar a integração com o Shifts no Teams, no Outlook e no Google agenda.

#### <a name="do-i-need-to-train-users-on-how-to-use-shifts-if-theyre-already-familiar-with-staffhub"></a>Preciso treinar os usuários sobre como usar Shifts se eles já estiverem familiarizados com a StaffHub?

Não é necessário nenhum treinamento adicional. Eles podem usar o Shifts da mesma forma que usavam o StaffHub. 
 
#### <a name="what-admin-controls-are-available-to-manage-whether-users-in-my-organization-can-use-shifts"></a>Quais controles de administrador estão disponíveis para gerenciar se os usuários em minha organização podem usar ou não o Shifts?  
  
Você pode ativar ou desativar o Shifts para todos os usuários da organização. No centro de administração do Microsoft Teams, você pode usar a política de configuração do aplicativo do Firstline Worker para personalizar o Teams para a sua Firstline Workforce. A política é configurada com aplicativos de Atividade, Turnos, Chat e Chamadas. Atribuir a política fixa esses aplicativos à barra de aplicativos no Teams para acesso rápido e fácil dos usuários. 

Para saber mais, confira [Gerenciar o aplicativo Shifts para sua organização no Teams](manage-the-shifts-app-for-your-organization-in-teams.md).

#### <a name="what-powershell-controls-are-available-for-teams"></a>Quais controles do PowerShell estão disponíveis no Teams

Gerencie o Teams por meio do módulo do PowerShell do Microsoft Teams e do módulo do PowerShell do Skype for Business. Para saber mais, confira [Visão Geral do PowerShell do Teams](../../teams-powershell-overview.md). No momento, não há cmdlets específicos do PowerShell disponíveis para o gerenciamento do Shifts.  

Se for algo de que você está interessado, envie-nos um pedido pelo [Portal de comentários do Teams no UserVoice](https://microsoftteams.uservoice.com/forums/555103-public-preview/category/182881-developer-platform).

#### <a name="what-will-happen-with-the-staffhub-api-which-is-currently-in-private-preview"></a>O que acontecerá com a API StaffHub que está na visualização privada?

Se você tiver criado todas as integrações com as APIs StaffHub, essas APIs serão substituídas. Lançamos as APIs do Microsoft Graph para dar suporte a integrações com as soluções de gerenciamento de recursos humanos, folha de pagamento ou força de trabalho. Para saber mais, confira [Usar a API do Microsoft Graph para trabalhar com o Teams](https://docs.microsoft.com/graph/api/resources/teams-api-overview).

#### <a name="is-reporting-available-to-view-shifts-usage-across-my-organization"></a>Os relatórios estão disponíveis para exibir o uso do Shifts na minha organização?

Como o Shifts é um aplicativo do Teams, você pode ver o uso por meio de relatórios do Teams. Para obter mais informações, confira [Relatórios do Teams no centro de Administração do Microsoft Teams](../../teams-analytics-and-reports/teams-reporting-reference.md) e [Relatórios de atividades do Teams no centro de administração do Microsoft 365](../../teams-activity-reports.md).

#### <a name="where-can-i-provide-feedback"></a>Onde posso enviar comentários?
  
No aplicativo móvel, toque em **Mais** no canto superior esquerdo da tela, vá para **configurações** > **Enviar comentários** ou **Relatar um problema**. Da mesma forma, nos aplicativos da área de trabalho e da Web, no canto inferior esquerdo da tela, vá para **Ajuda** > **Compartilhar uma ideia** ou **Relatar m problema**.
