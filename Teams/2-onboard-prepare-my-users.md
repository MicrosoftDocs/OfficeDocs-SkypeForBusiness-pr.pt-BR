# <a name="prepare-my-users"></a>Preparar meus usuários

Este artigo fornece uma visão geral dos requisitos de Preparando os usuários para utilizar serviços de nuvem de voz. Por seus usuários a preparação adequada, você vai Certifique-se de que está sendo fornecido o valor máximo para sua organização.

<!--ENDOFSECTION-->

## <a name="preparing-for-user-adoption"></a>Preparação para a adoção de usuário

Após você ter protegido sua equipe de projeto e definido o escopo e os objetivos do seu projeto, a próxima etapa é garantir organizacional e preparação do usuário, uma atividade que você concluir em paralelo com sua prontidão técnica. Para compreender o valor de sua implantação, os usuários devem usar realmente e adotá-lo. Habilitar simplesmente uma solução não garante que você vai atingir suas metas. Além disso, os usuários têm diferentes casos de uso e os estilos de aprendizagem variados e adaptar-se para uma nova tecnologia em velocidades diferentes. A boa notícia é que o gerenciamento de alterações não é muito complicado, mas exige um esforço concentrado. As discussões guiadas incluídas abaixo são projetadas para ajudá-lo a compreender sua base de usuários e a preparar o nível adequado de educação para facilitar e acelerar a adoção dos usuários.

Atividade de proprietário: Gerenciador de alteração de adoção como líder, líder, patrocinador executivo, gerentes de treinamento/Marketing/suporte do Project

> [!NOTE] 
> As atividades de avaliação fornecidas abaixo são adaptáveis a iniciativa qualquer alteração em sua organização. Endereços simplesmente as perguntas com base no escopo do seu projeto. A discussão destas atividades, "nova solução" pode aplicar a chamada de áudio, conferência de áudio ou a atualização do Skype para a equipes de negócios.

<!--ENDOFSECTION-->

## <a name="create-your-vision-to-value-statement"></a>Criar sua instrução vision-valor

Definir sua visão de projeto habilita os acionistas chaves para exibir uma visão geral. Geralmente, isso corresponde a fatores comerciais&mdash;como economia de custos, aumento da produtividade e eficiência operacional&mdash;e se alinha com as metas definidas anteriormente no projeto. Para um usuário, o mesmo projeto poderia ser exibido como a possibilidade de eliminar ferramentas familiares, interrupção de produtividade e falta de compaixão pelo gerenciamento. Demorar para colocar-se no lugar dos usuários e converter sua visão do projeto em uma instrução de valor que será atraente para acessá-los. Empregar esta declaração de valor em suas comunicações e atividades de treinamento para capacitar os usuários para ver como as equipes podem trabalhar para eles.

## <a name="determine-organizational-change-readiness"></a>Determinar a preparação de alteração organizacional

Prontidão organizacional ajuda a determinar como receptivos (ou não) os usuários podem ter e alterando a maneira como eles funcionam. Saber como os usuários podem reagir para alterar capacita a sua organização proativamente preocupações, ajustar seu plano de distribuição para recepção ideal e identificar os usuários que podem se tornar seu campeões ou ambassadors para ajudar a facilitar a alteração.

### <a name="what-percent-of-users-fall-into-each-bucket"></a>Que porcentagem de usuários se enquadram em cada no processo de Balde?

| **Tipo de usuário**                                                             | **%** |
|------------------------------------------------------------------------------|-------|
| *Primeiros usuários* solicitar a solução antes que ele esteja disponível.                  |       |
| *Informados usuários* aceitar a solução assim que seu valor é demonstrado.    |       |
| *Retardatários* rejeitar a solução, mesmo quando enviada em alterar.                 |       |


> [!TIP]
> Primeiros usuários são grandes candidatos para a implantação do piloto e também as iniciativas dos campeões. Se você tiver uma grande porcentagem de retardatários, demore para focalizar em mensagens para os usuários e de suporte ajudar a conectá-los bordo com a alteração do valor.

### <a name="what-percent-of-users-fall-into-each-competency"></a>Que porcentagem de usuários se enquadram em cada competência?

| **Tipo de aprendizado rápido**                                                        | **%** |
|----------------------------------------------------------------------------|-------|
| Alunos *motivados* check-out de recursos de pesquisa, saiba fazendo.                         |       |
| *Equipe players* gostam de grupo e treinamento interativo; eles irão junto com os colegas de trabalho. |       |
| Em busca de *suporte* espera "luva de branca" ou assistência pessoal durante.                   |       |

 
> [!TIP]
> Aprendizado não é única. Compreendendo como seus usuários a aprender melhor, você pode criar um programa de treinamento robusta para ajudar a acelerar seu aprendizado. Se muitos usuários são motivados, vincular vídeos e guias de usuário podem ser suficiente. Para aqueles que precisam de atenção especializada, certifique-se de agendar algum tempo pessoal durante&mdash;ou inscrever-se os representantes para ajudar os usuários sobre o aprendizado.

## <a name="assess-your-user-base"></a>Avaliar a sua base de usuários

Análise de nível de pessoas, oferece compreensão da sua personagens do usuário, os colaboradores e os casos de uso. Tempo empregado no entender quem são os usuários e como eles funcionam, você pode garantir que eles têm o que precisam ser bem-sucedida à medida que implementa a alteração. Isso pode incluir mensagens direcionado, treinamento ou até mesmo dispositivos como headsets e webcams. Por exemplo, um executivo era esperado treinamento individuais, enquanto os representantes de campo em viagem pode se beneficiar um vídeo, que eles podem assistir em seu tempo livre de treinamento. Além disso, sua análise de nível de pessoas pode ajudar a identificar quem deve começar a usar a nova funcionalidade hoje versus que podem ser mais bem atendidas em espera.

Para cada grupo de pessoa, abordam as seguintes com sua equipe de projeto:

-   Quais são as características principais desse grupo (ou seja, o que é um "dia da vida útil" normal)?

-   Como esse grupo serão beneficiados com a nova solução? Quais recursos e funcionalidades será necessário para serem eficientes?

-   Quais dispositivos eles precisará (por exemplo, fones de ouvido ou alto-falantes), e eles já tiver em-los?

-   Quais mensagens de valor adicional, se houver, é necessário para criar desejo nesse grupo começar a usar a nova solução?

-   Qual é a melhor forma para este grupo receber:

    -   Divulgação (por exemplo, email, cartazes, gerentes de departamento)?

    -   Treinamento (por exemplo, treinamento online, guia do usuário, luva de branca)?

    -   Suporte (autoatendimento, assistência técnica, campeões)?  

-   Este grupo é um bom candidato para um programa piloto ou antecipado adotar? Em caso afirmativo, faça um plano para obter apoio da direção desses usuários participem do piloto. Para obter mais informações sobre como implementar um programa piloto bem-sucedida, consulte [Equipes piloto junto com Skype para negócios](pilot-essentials.md).

-   Quais são as características principais para este caso de uso/cenário?

-   O que é uma boa sobre a solução atual?

-   Que desafios, se houver, enfrentados com sua solução atual?

-   Como será a nova solução otimizar a experiência do usuário?

## <a name="optimize-your-readiness-plan"></a>Otimize seu plano de preparação
Depois que você entende quem são os usuários e como eles serão funcionam na nova solução, você pode projetar seu plano de preparação. Seu plano de preparação do usuário incorpora como você vai se comunicar, treinar e suporte a seus usuários como você piloto, integrado e operar a nova solução. À medida que você descobertas nas etapas anteriores neste artigo, não é única preparação. Comece com um amplo número-alcance divulgação e plano de treinamento, incorporando o valor de mensagens são definidas no início deste processo. Em seguida, ajustar suas mensagens, treinamento e suporte ao personagens ou colaboradores que você definiu, conforme apropriado. Seu plano personalizado permitirá mais rapidamente se relacionam com a nova solução durante a demonstração que você entende suas necessidades, que ajudarão a facilitar a transição de seu usuários.
