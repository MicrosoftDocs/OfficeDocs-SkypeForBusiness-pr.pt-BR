# <a name="configure-networking"></a>Configurar a rede

| Não | Atividade ou tarefa | Descrição | Concluído? | Informações adicionais |
|----|--------------------------------------------------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|------------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| 1  | Examinar os requisitos de rede para equipes | Ter uma visão geral das suas necessidades de rede antes de ir para os detalhes de redes. | | [Preparar a rede da organização para o Microsoft Teams](https://docs.microsoft.com/microsoftteams/prepare-network)                                                               |
| 2  | Entregar o workshop de preparação da rede | Conclua o workshop de preparação da rede de MyAdvisor e trabalhar com suas equipes a identificar áreas de interesse para endereço como parte do esforço de remediação. | | [Ativos de preparação de rede MyAdvisor](https://myadvisor.fasttrack.microsoft.com/CloudVoice/Downloads?SelectedIDs=4_3_0_2,4_3_0_3,4_3_0_5,4_3_0_6,4_3_0_7,4_3_0_8,4_3_0_10,4_3_0_11) |
| 3  | Use o Planejador de rede | Use o Planejador de rede para calcular a largura de banda necessária para todos os sites no escopo e locais. | | [Planejador de rede MyAdvisor](https://myadvisor.fasttrack.microsoft.com/CloudVoice/NetworkPlanner)                                                                                   |
| 4  | Validar o tamanho de pool NAT necessário para conectividade de usuário | Certifique-se de que adequada endereços IP públicos são atribuídos para os pools NAT para evitar o esgotamento de porta. Esgotamento de porta será contribuir para usuários internos e dispositivos sendo Impossível conectar ao serviço do Office 365. <br/><br/>Problemas de conectividade são a causa principal dos problemas de percepção do usuário para os serviços de nuvem. | | [Suporte NAT com o Office 365](https://support.office.com/article/NAT-support-with-Office-365-170e96ea-d65d-4e51-acac-1de56abe39b9) |
| 5  | Implementar o roteamento mais eficiente em centros de dados da Microsoft | Identifica os locais que podem usar pontos de saída locais ou regionais para se conectar à rede Microsoft de maneira mais eficiente possível. <br/><br/>O artigo na coluna **informações adicionais** descreve como os clientes podem se beneficiar dos recursos na resolução de nomes do Office 365 e roteamento IP para ser conectado com eficiência com o datacenter regional mais próximo. | | [Conectividade de cliente do Office 365](https://support.office.com/article/Client-connectivity-4232abcf-4ae5-43aa-bfa1-9a078a99c78b) |
| 6  | Configurar portas de firewall solicitadas para conectividade com equipes | Revise IPs e URLs do Office 365 para identificar e testar as portas de firewall necessárias para a conectividade entre clientes locais e os servidores e serviços do Office 365. <br/><br/>Para um ambiente com suporte, você deve abrir todas as portas nos seus firewalls. Falha ao abrir alguns ou intervalos de portas negativamente afetará a experiência do usuário. Configure portas de mídia em seus firewalls com base nas orientações na coluna **informações adicionais** . | | [URLs do Office 365 e endereços IP – equipes da Microsoft](https://support.office.com/article/office-365-urls-and-ip-address-ranges-8548a211-3fe7-47cb-abb1-355ea5aa88a2#bkmk_teams) |
| 7  | Configurar servidores proxy | Configure o ambiente para que servidores proxy são ignorados e você pode conectar os usuários diretamente para o Office 365 usando UDP, para a qualidade áudio e vídeo melhor. Quando a mídia em tempo real é forçada a atravessar um servidor proxy, a pilha de mídia em equipes é forçada a failback para TCP, que afeta negativamente qualidade. <br/><br/>Para a experiência do usuário mais alta qualidade, sempre preferem UDP sobre TCP. | | [Planejamento de gerenciamento de serviço e de qualidade](https://docs.microsoft.com/MicrosoftTeams/envision-planning-for-service-management-and-quality-complete-guide) |
| 8  | Configurar split túnel VPN | É recomendável que você forneça um caminho alternativo para o tráfego de equipes que ignora a VPN, comumente conhecida como *divisão de túnel VPN*. Túnel em divisão significa que o tráfego para o Office 365 não atravessa VPN mas vai diretamente para o Office 365. Essa alteração tem um impacto positivo sobre a qualidade, mas também apresenta um benefício secundário de reduzir a carga dos dispositivos de VPN e da rede da organização. | | Para implementar uma VPN de divisão de túnel, consulte seu fornecedor VPN para obter informações detalhadas de configuração. |
| 9  | Configurar a priorização de pacote usando o QoS | QoS deve ser implementada em todos os segmentos de uma rede gerenciada. Mesmo quando uma rede tenha sido adequadamente provisionada para a largura de banda, QoS fornece mitigação de riscos no caso de eventos de rede inesperadas. Quando a QoS é implementado, o tráfego de voz é priorizado para que esses eventos inesperados negativamente não afetam a qualidade. | | [Planejamento de gerenciamento de serviço e de qualidade](https://docs.microsoft.com/MicrosoftTeams/envision-planning-for-service-management-and-quality-complete-guide) <br/><br/>[Qualidade de Serviço no Microsoft Teams](https://docs.microsoft.com/MicrosoftTeams/qos-in-teams) |
| 10 | Otimizar redes Wi-Fi de qualidade e desempenho | Vários fatores entram em cena quando você otimizar sua rede Wi-Fi: <ul><li>Implementação de QoS ou Wi-Fi WMM (multimídia) para garantir que o tráfego de mídia através das redes Wi-Fi priorizado.</li><li>Posicionamento de ponto de planejamento e otimização faixas Wi-Fi e acesso. O intervalo de 2,4 GHz pode fornecer um desempenho adequado, dependendo de posicionamento de ponto de acesso.</li></ul> Consulte seu fornecedor de Wi-Fi para obter orientações específicas. | | [Recomendações de Wi-Fi para pontos de extremidade](https://docs.microsoft.com/MicrosoftTeams/envision-planning-for-service-management-and-quality-complete-guide#wi-fi-recommendations-for-endpoints) |
| 11 | Validar a conectividade de rede usando a ferramenta de avaliação de rede | Use a ferramenta de avaliação de rede para Skype para equipes e de negócios para testar a conectividade com todos os endereços IP e portas usadas no Skype para reuniões e chamadas Business Online e equipes. Baixe a ferramenta e consulte Usage.docx para obter detalhes sobre como usar a ferramenta e interpretar os resultados do teste. Recomendamos que você execute a ferramenta de um PC cliente em cada local onde as equipes serão usadas. | | [Skype para equipes ferramenta de avaliação de rede e de negócios](https://go.microsoft.com/fwlink/?linkid=855799) |
