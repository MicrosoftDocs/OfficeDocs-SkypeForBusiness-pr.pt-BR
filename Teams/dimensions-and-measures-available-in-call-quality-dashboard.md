# <a name="dimensions-and-measures-available-in-call-quality-dashboard"></a>Dimensões e medidas disponíveis no Painel de Qualidade de Chamadas

O painel de qualidade de chamada (CQD) para Microsoft Teams e o Skype for Business Online permite que você obtenha ideias sobre a qualidade das chamadas feitas usando o Microsoft Teams e os serviços do Skype for Business. Este tópico fornece informações detalhadas sobre as dimensões e medidas expostas por CQD. Para saber mais sobre o CQD e como habilitá-lo, confira [ativar e usar o painel de qualidade de chamada para Microsoft Teams e Skype for Business online](turning-on-and-using-call-quality-dashboard.md).



## <a name="first-and-second-endpoint-classification"></a>Classificação de primeiro e segundo pontos de extremidade

Muitas das dimensões e medidas no painel de qualidade de chamada (CQD) são rotuladas como primeiro ou segundo. A lógica a seguir determina qual ponto de extremidade envolvido no fluxo ou chamada é rotulado como primeiro:

- O primeiro será sempre um ponto de extremidade do servidor (AV MCU, servidor de mediação, etc.) se um servidor estiver envolvido no fluxo/chamada.

- Segundo será sempre um ponto de extremidade do cliente, a menos que o fluxo esteja entre dois pontos de extremidade do servidor.

- Se os dois pontos de extremidade forem do mesmo tipo, o pedido que é primeiro versus segundo baseia-se na ordenação interna da categoria do agente do usuário. Isso garante que a solicitação seja consistente.

### <a name="notes-on-dimension-data-typeunits"></a>Observações sobre tipo/unidades de dados de dimensão:

#### <a name="range"></a>Gama

As dimensões que são fornecidas como intervalo ou grupo de valores são mostradas usando o seguinte formato:

 _\<Cadeia de caracteres\> de\<ordem de classificação\> - \<[limite superior\>inclusivo inferior, Exclusive)_

Por exemplo, a dimensão duração (minutos) representa a duração da chamada em segundos com o valor informado como um intervalo de valores.

|||
|:-----|:-----|
|**Duração (minutos)** <br/> |**Como interpretar** <br/> |
|062: [0-0)  <br/> |Duração do fluxo = 0 minutos  <br/> |
|064: [1-2)  <br/> |1 minuto < = duração do fluxo < 2 minutos  <br/> |
|065: [2-3)  <br/> |2 minutos < = duração do fluxo < 3 minutos  <br/> |
|066: [3-4)  <br/> |3 minutos < = duração do fluxo < de 4 minutos  <br/> |
|…  <br/> |…  <br/> |

O <sort order string> é usado para controlar a ordem de classificação ao apresentar os dados e pode ser usado para filtragem. Por exemplo, um filtro sobre Duration (minutos) < "065", mostraria fluxos com duração inferior a 2 minutos (o "0" à esquerda é necessário para o filtro funcionar como esperado).

> [!NOTE]
> O valor real da cadeia de caracteres da ordem de classificação não é significativo.

#### <a name="enumeration-pair"></a>Par de enumeração

As dimensões que são fornecidas como um par de enumeração são mostradas usando o seguinte formato:

 _\<valor de enumeração de um ponto\> de \<extremidade: valor de enumeração do outro ponto de extremidade\>_

A ordem dos valores de enumeração é consistente, mas não reflete a ordem dos primeiros ou dos segundos pontos de extremidade.

Por exemplo, o par de detalhes de conexão de rede mostra os valores de detalhes de conexão de rede para os dois pontos de extremidade:

|||
|:-----|:-----|
|**Par de detalhes de conexão de rede** <br/> |**Como interpretar** <br/> |
|Wired: Wired  <br/> |O primeiro e o segundo pontos de extremidade usavam conexões Ethernet com fio.  <br/> |
|Com fio: WiFi  <br/> |O primeiro ponto de extremidade usou conexão Ethernet com fio e o segundo ponto de extremidade usou conexão Wi-Fi, ou o segundo ponto de extremidade usou conexão Ethernet com fio e o primeiro ponto de extremidade usou conexão Wi-Fi.  <br/> |
|: WiFi  <br/> |O primeiro ponto de extremidade usou uma conexão WiFi e a conexão de rede usada pelo segundo ponto de extremidade é desconhecida ou o segundo ponto de extremidade usou uma conexão WiFi e a conexão de rede usada pelo primeiro ponto de extremidade é desconhecida.  <br/> |
|…  <br/> |…  <br/> |

#### <a name="blank-values"></a>Valores em branco

A tabela acima lista os possíveis motivos pelos quais uma dimensão pode estar em branco. Muitas dimensões e medidas ficarão em branco se a dimensão de registro de QoE disponível for falsa. Isso geralmente ocorre quando a chamada não foi estabelecida com sucesso.
