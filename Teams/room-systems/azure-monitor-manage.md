# <a name="manage-microsoft-teams-rooms-devices-with-azure-monitor"></a>Gerencia dispositivos de salas de equipes da Microsoft com Monitor do Azure

Este artigo discute como gerenciar os dispositivos de salas de equipes da Microsoft de forma integrada e de ponta a ponta, usando o Monitor do Azure.

Você pode configurar o Azure Monitor para fornecer telemetria básicas que irão ajudar você a gerenciar Skype reunião dispositivos de sala (consulte [gerenciamento de planejar salas de equipes da Microsoft com o Azure Monitor](azure-monitor-plan.md) e [gerenciamento de implantar o Microsoft equipes salas com Monitor do Azure]((azure-monitor-deploy.md) for details). Como sua solução de gerenciamento for envelhecendo, você pode usar recursos de gerenciamento e dados adicionais para criar uma visão mais detalhada do desempenho do dispositivo.

## <a name="understand-the-log-entries"></a>Entenda as entradas do log

Descrições de evento a seguir são inseridas no campo de descrição do log de eventos a cada cinco minutos, quando o aplicativo Microsoft equipes sala registra as informações correspondentes no log de eventos do Windows. O agente de monitoramento do Microsoft passa esses registros para análise de Log no Monitor do Azure, que analisa-los ao painel de controle que você criou no [gerenciamento de implantar o Microsoft equipes salas com Monitor do Azure](azure-monitor-deploy.md). No painel, você pode examinar as entradas individuais do log para estabelecer sequências de ações, se necessário. 

As IDs do Evento 2000 e 3000 indicam que o dispositivo em questão está funcionando conforme esperado. As IDs do Evento 2001 e 3001 indicam que ocorreu um problema. A ID do Evento 4000 pode exigir ações, caso seja observada com mais frequência que o esperado em comparação com uma linha de base que você define ou com outros dispositivos implantados em sua organização.

Se você entender essas descrições de eventos, será alertado rapidamente sobre problemas e terá um ponto inicial para a solução de problemas.
