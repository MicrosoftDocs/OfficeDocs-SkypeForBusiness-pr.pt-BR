# <a name="implement-your-move-to-microsoft-teams"></a>Implementar sua mudança para o Microsoft Teams

Este artigo fornece suas diretrizes sobre implantação e implementação do Microsoft Teams para os seguintes cenários:

1. Para **organizações que já usam o Microsoft Teams**
2. Para **organizações que ainda não usam o Microsoft Teams**

## <a name="deploying-and-implementing-microsoft-teams-for-organizations-already-using-microsoft-teams"></a>Implantando e implementando o Microsoft Teams para organizações que já usam o Microsoft Teams
 
Antes de prosseguir, confirme se você concluiu as seguintes atividades: 

- Trabalho em equipe previsto para sua organização  
- Campeões identificados e participantes críticos 
- Definiu o escopo do seu projeto  
- Microsoft Teams pilotado 
- Preparou sua organização 

Siga as diretrizes neste artigo, se você já tiver o **Microsoft Teams** lançado e quiser mover seus usuários licenciados do Kaizala para o Microsoft Teams. 
   
Recomendamos a seguinte estrutura para transição:  
   
**Planejar sua transição** 
   
1. Primeiro, mova seus grupos de chat para o Microsoft Teams.
1. Em seguida, use aplicativos no Teams para substituir cartões de ação do Kaizala.
1. Para garantir que os funcionários de linha de frente sejam equipados, atualize para a funcionalidade do Teams Frontline Worker. Para ver um guia de início rápido sobre o Teams Frontline Worker, consulte Guia de Início Rápido do Trabalho de Linha [de Frente.](./flw-quickstart.yml)
1. Considere a integração de fornecedores e parceiros para convidar como convidados no Teams.  
  
**Gerenciar alterações**  
   
1. Crie uma campanha de comunicação para conscientizar o projeto de transição. 
1. Notifique sua organização sobre as alterações e identifique os campeões para impulsionar a adoção do Microsoft Teams. 
1. Capacitar os usuários a saber mais sobre o Microsoft Teams por meio de treinamento gratuito. 
   
Para saber mais, confira [Treinamento em vídeo do Microsoft Teams.](https://support.microsoft.com/office/microsoft-teams-video-training-4f108e54-240b-4351-8084-b1089f0d21d7?ui=en-us&rs=en-us&ad=us)   
 
## <a name="deploying-and-implementing-microsoft-teams-for-organizations-not-yet-using-microsoft-teams"></a>Implantando e implementando o Microsoft Teams para organizações que ainda não usam o Microsoft Teams
 
Siga as diretrizes nesta seção, se você estiver usando o Kaizala e quiser mover seus usuários do Kaizala para o Microsoft Teams.
   
**Antes de prosseguir, confirme se você concluiu as seguintes atividades:** 
   
- Trabalho em equipe previsto para sua organização 
- Campeões identificados e participantes críticos 
- Definiu o escopo do seu projeto  
- Microsoft Teams pilotado
- Preparou sua organização  
   
Recomendamos a seguinte estrutura para transição: 
   
- **Vislumbrar o trabalho em equipe para sua organização** 
   
   Listar os cenários atuais em que o Kaizala é usado. Em seguida, visualize cenários acima e além com o Microsoft Teams.  

- **Equipes piloto**

   Lançar o Teams para um grupo piloto de usuários com um conjunto de cenários priorizado. 

- **Implantar o Teams** 

   Aprenda com o grupo piloto. Prepare-se para rolar para a organização completa.  

- **Usar o Kaizala e o Teams**  

   Para encontrar o Microsoft Teams certo para sua empresa, consulte [Comparar opções do Microsoft Teams Online | Microsoft Teams](https://www.microsoft.com/microsoft-teams/compare-microsoft-teams-options). 

- **Gerenciar alterações** 

   Prepare sua organização para mudanças por meio do treinamento do usuário final para impulsionar a adoção. Os administradores e campeões de IT podem impulsionar os esforços para influenciar o gerenciamento de alterações positivos em movimento.  

- **Planejar sua adoção do Teams**

    Prepare-se para mover soluções personalizadas do Kaizala para o Teams, por exemplo, cartões de ação personalizados. 
     
- **Mover sua organização para o Teams** 

    Capacitar sua organização com uma ferramenta simplificada para comunicação do CEO para a linha de frente!
